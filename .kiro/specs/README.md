## About Kiro Specs

Kiro specs use a structured approach to feature development:
1. **Requirements** - What needs to be built
2. **Design** - How it will be built
3. **Tasks** - Step-by-step implementation

Once features are complete, their documentation moves to `/docs` and specs are archived here for reference.
